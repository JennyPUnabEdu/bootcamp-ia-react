Bloque 1: Documentación General

1.1 Introducción

Bootcamp 360 Plataforma Cursos - IA es una plataforma web educativa de alto nivel, interactiva y modular, diseñada para la enseñanza de herramientas de Inteligencia Artificial Generativa (ChatGPT y Google Gemini) a estudiantes de ingeniería. Este proyecto fue desarrollado en el contexto de una práctica profesional para la Facultad de Ingeniería de la UNAB.

La plataforma moderniza la experiencia de aprendizaje, reemplazando las presentaciones estáticas tradicionales por un entorno web gamificado que simula interacciones reales con IA.

1.2 Objetivos

Objetivo General

Desarrollar una plataforma web interactiva que permita a los estudiantes experimentar y comprender el funcionamiento de las IAs generativas mediante simulaciones prácticas y contenido dinámico.

Objetivos Específicos

  • Simulación Realista: Interfaces de chat simuladas para practicar Prompt Engineering.
  • Interactividad: Sistema de slides dinámicas (Slide Engine).
  • Evaluación Automatizada: Quizzes y rúbricas con feedback inmediato.
  • Acceso a Recursos: Centralización de material de estudio.

1.3 Alcance del Proyecto

El sistema abarca una suite de aplicaciones "Single Page Application" (SPA) integradas bajo un mismo enrutador. Incluye:

  • Motor de Presentación JSON-driven.
  • Módulos temáticos independientes.
  • Modo Versus (Comparativas IA).
  • Sistema de Checklists y Rúbricas.

1.4 Público Objetivo

  • Estudiantes de Ingeniería (UNAB): Usuarios finales.
  • Cuerpo Docente: Apoyo en clases presenciales o remotas.

1.5 Stack Tecnológico

  • Frontend: React 19 + Vite 7
  • Estilos: Tailwind CSS 4
  • Routing: React Router DOM 7
  • Servidor: Apache 2 (Ubuntu AWS EC2)

1.6 Arquitectura General

La arquitectura es una SPA servida estáticamente. Frontend-First: La lógica reside en el cliente. Data-Driven: Contenido desacoplado. Sub-Directory: Despliegue en /curso-ia.

1.7 Diagrama Lógico

  1. Browser: Usuario pide URL.
  2. AWS EC2 (Apache): Recibe petición HTTP.
  3. Routing Servidor: Si URL inicia con /curso-ia, sirve index.html React.
  4. React Init: Carga JS/CSS. React Router toma el control.
  5. Render: Generación dinámica de vista.

1.10 Roles y Responsabilidades

Rol Responsabilidad Perfil
SysAdmin Mantenimiento servidor, Apache, DNS, Backups. DevOps / Infra.
Dev Frontend Mantenimiento React, nuevos módulos, bugs. Dev React Semi-senior.
Editor Edición de archivos JSON en src/data/. Ayudante / Profesor.

1.11 Requisitos No Funcionales

  • Disponibilidad: 24/7.
  • Rendimiento: Carga < 3 segundos en 4G.
  • Compatibilidad: Chrome, Firefox, Edge, Safari.
  • Escalabilidad: Arquitectura modular para N módulos.
  • Seguridad: Stateless (sin datos sensibles de usuario).

1.12 Limitaciones Conocidas

Persistencia: Al ser una SPA stateless, el progreso se pierde al borrar caché, salvo implementación futura de LocalStorage/Firebase.

Conexión: Requiere internet constante. Simulación: Los chats son pre-programados, no consultan APIs reales.

Bloque 2: Manual de Usuario

Bienvenido a la Plataforma de Aprendizaje IA. Este manual le guiará en el uso de la herramienta.

2.3 Cómo acceder

  1. Abra su navegador web (Chrome/Edge recomendado).
  2. Ingrese a: proyectos.icinf.icfunab.cl/curso-ia
  3. Verá la pantalla de bienvenida.

2.4 Uso General

Inicio: Seleccione un módulo haciendo clic en "Iniciar Módulo".

Navegación: Use las flechas laterales o los botones en pantalla para avanzar/retroceder. Use el icono de "Casa" para volver al menú.

2.6 Tipos de Diapositivas

  • Informativas: Texto e imágenes estáticas.
  • Simulaciones (Chat): Escriba o presione "Run" para interactuar con la IA simulada.
  • Comparativas: Deslice controles para ver diferencias entre modelos.

2.8 Evaluaciones

Responda los "Momentos de Quiz" seleccionando la opción correcta. Recibirá feedback inmediato sobre su respuesta.

2.9 Solución de Problemas

  • Pantalla blanca: Recargue la página (F5).
  • No avanza: Complete la interacción requerida en la slide.
  • Error 404: Vuelva a la raíz /curso-ia.

Bloque 3: Manual de Analista / Técnico

3.1 Arquitectura Frontend (Data-Driven)

El proyecto separa lógica de contenido:

  • Datos: src/data/ (Arrays de objetos JSON).
  • Lógica: Componentes iteran sobre los datos para renderizar.

Routing: Se utiliza react-router-dom con un basename configurado para el subdirectorio.

3.3 Build & Producción

Vite Config: Propiedad base: '/curso-ia/' obligatoria.

Comando de compilación: npm run build. Genera la carpeta dist/.

3.4 Configuración Apache (.htaccess)

CRÍTICO: El archivo .htaccess es obligatorio en la carpeta pública para evitar errores 404 al recargar páginas internas.

Ubicación: /var/www/proyectos.icinf.icfunab.cl/html/curso-ia/.htaccess

<IfModule mod_rewrite.c>
  RewriteEngine On
  # Base para reglas de reescritura
  RewriteBase /curso-ia/
  
  # Si el archivo existe físicamente, servirlo
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  
  # Si no, redirigir a index.html (Client-Side Routing)
  RewriteRule . /curso-ia/index.html [L]
</IfModule>

3.5 Git & Mantenimiento

PELIGRO: NUNCA ejecutar git pull en la carpeta pública (html/curso-ia). Esto mezclará código fuente con código compilado.

Flujo Correcto:

  1. Ir a carpeta fuente: cd /var/www/curso-ia
  2. git pull
  3. npm run build
  4. Copiar dist: cp -r dist/* [carpeta_publica]

Bloque 4: Despliegue y Operación

4.1 Flujo Completo de Despliegue

Siga estos pasos exactos para actualizar el sitio:

1. Actualización de Código

ssh ubuntu@
cd /var/www/curso-ia
git pull origin main
npm install  # (Solo si cambiaron dependencias)

2. Compilación

npm run build

Esto actualiza la carpeta local dist/.

3. Publicación

# Copiar contenido de dist a la carpeta web de Apache
sudo cp -r dist/* /var/www/proyectos.icinf.icfunab.cl/html/curso-ia/

4. Permisos (Vital)

sudo chown -R www-data:www-data /var/www/proyectos.icinf.icfunab.cl/html/curso-ia
sudo chmod -R 755 /var/www/proyectos.icinf.icfunab.cl/html/curso-ia

Justificación: Apache (www-data) necesita ser dueño para servir archivos. 755 permite lectura pública pero solo escritura por el dueño.

4.3 Troubleshooting

  • Cambios no visibles: Borrar caché del navegador. Verificar que se ejecutó el paso de copia (cp).
  • Error 404: Verificar existencia de .htaccess oculto en carpeta pública.
  • Forbidden (403): Re-aplicar comandos de permisos.

4.5 Respaldo (DR)

Fuente: Repositorio GitHub es la verdad única. Recuperación: Clonar repo + npm run build (RTO < 30 min).

React Router
Librería estándar para la navegación en aplicaciones React.
VirtualHost
Configuración de Apache para servir múltiples dominios desde un mismo servidor.
Cache Busting
Técnica de agregar hashes a los nombres de archivo para forzar la actualización en el navegador.
Deploy
Proceso de llevar el código a producción.
EC2
Servicio de servidores virtuales de AWS.
Prompt Engineering
Arte de diseñar instrucciones efectivas para guiar modelos de IA.
